# Docs Contribution Notes

## Writing style

- Keep explanations practical and product-oriented
- Use StreakBit terminology consistently
- Prefer short sections with clear headings

## Areas that need documentation

- Real wallet integration once live Stacks contract calls replace demo state
- Production deployment steps for mainnet
- SDK usage once the separate `streakbit-sdk` repo is created

## Validation

When docs touch commands or workflows, verify them locally if possible before merging.
